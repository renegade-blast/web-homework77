## 一、修复前漏洞清单

| # | 漏洞 | 严重程度 | 说明 |
|---|------|---------|------|
| 1 | **SSRF 内网访问** | 🔴 高危 | 可访问 `http://127.0.0.1:5000/`（自身服务）、`http://192.168.x.x/`（内网） |
| 2 | **file:// 协议任意文件读取** | 🔴 高危 | `file:///etc/passwd`、`file:///app.py` 可读取任意文件 |
| 3 | **ftp:// 协议端口扫描** | 🟡 中危 | `ftp://192.168.1.1:21` 可探测内网服务 |
| 4 | **gopher:// 协议 SSRF 增强** | 🟡 中危 | 可攻击 Redis 等服务 |
| 5 | **超长 URL 日志注入** | 🟢 低危 | 10k URL 完整写入日志文件 |

## 二、修复措施

### 2.1 协议白名单

```python
# ✅ 修复后：只允许 http 和 https
if parsed.scheme not in ("http", "https"):
    return False
```

**拦截的协议**: `file://`、`ftp://`、`gopher://`、`data:`、`javascript:` 等

### 2.2 内网地址黑名单

```python
# ❌ 修复前：不检查目标地址
resp = urllib.request.urlopen(url)  # 可访问 127.0.0.1

# ✅ 修复后：禁止所有内网地址
禁止列表:
  - localhost, 127.0.0.1, 0.0.0.0, ::1        # 回环地址
  - 10.x.x.x                                   # A 类私有
  - 172.16-31.x.x                              # B 类私有
  - 192.168.x.x                                # C 类私有
  - 169.254.x.x                                # 链路本地
  - 100.64-127.x.x                             # CGNAT
```

### 2.3 URL 日志截断

```python
# ❌ 修复前：完整记录
logger.info(f"URL抓取: {username} 请求 {url}")  # 10k URL 全量写入

# ✅ 修复后：截断至 200 字符
log_url = url[:200] + "..." if len(url) > 200 else url
logger.info(f"URL抓取: {username} 请求 {log_url}")
```
