# 头像上传功能 — 独立实现报告

> **对应需求**: 在登录、注册、搜索基础上增加头像上传功能
> **实现提交**: `7609933`
> **实现日期**: 2026年7月9日

---

## 一、需求概述

在已有功能基础上，新增用户头像上传功能，支持文件上传、保存、预览，不检查文件类型。

## 二、新增内容

### 2.1 后端路由

| 路由 | 方法 | 功能 |
|------|------|------|
| `/upload` | GET/POST | 上传页面 + 文件保存 |

### 2.2 存储配置

```python
app.config["MAX_CONTENT_LENGTH"] = 16 * 1024 * 1024  # 16MB
```

### 2.3 文件处理

- 保存到 `static/uploads/` 目录
- 使用用户提供的原始文件名
- 不检查文件后缀或 MIME 类型

## 三、安全缺陷

| 缺陷 | 说明 |
|------|------|
| 路径穿越 | `filename=../../evil.txt` 可逃逸 uploads 目录 |
| 存储型 XSS | 上传 `.html` 文件包含 `<script>` 可在同源执行 |
| 同名覆盖 | 同名文件静默覆盖 |
| 超长文件名 | 500 字符文件名导致崩溃 |

## 四、修复内容

| 缺陷 | 修复方式 |
|------|---------|
| 路径穿越 | `os.path.basename()` 截断 |
| 存储型 XSS | `.html` 文件重命名为 `.txt` |
| 同名覆盖 | `os.path.exists()` 检查 |
| 超长文件名 | 截断至 200 字符 + try/except |
